# 📊 Tech Store: Optimización de Reportabilidad y KPIs de Ventas

¡Bienvenido! Este proyecto presenta una solución integral de análisis de datos para una cadena de retail tecnológica en Chile, abarcando desde el modelado de datos en SQL hasta la comunicación de insights estratégicos.

## 🚀 Resumen del Proyecto
Transformé un conjunto de datos desorganizado en un sistema de Business Intelligence que permite monitorear el cumplimiento de metas de ventas ($3M por tienda) y el desempeño regional.

---

## 🛠️ Stack Tecnológico
* **Base de Datos:** SQL (Modelado Relacional Estrella).
* **Visualización:** Power BI (DAX para métricas de cumplimiento).
* **Análisis:** Data Storytelling y Diagnóstico Organizacional.
* **Entorno:** Google Colab

---

## 📈 Desafíos y Soluciones (Caso de Estudio)

### 1. El Problema (The Challenge)
La empresa presentaba una **brecha de sostenibilidad**: solo el 40% de las sucursales alcanzaba la meta de rentabilidad. Además, existía una desconexión lógica entre vendedores y sus zonas geográficas en la base de datos original.

### 2. La Solución Técnica
* **Normalización:** Diseñé un modelo entidad-relación que asegura la integridad referencial.
* **Dashboards:** Implementé visualizaciones con "Semáforos" (formato condicional) para identificar tiendas críticas como **Concepción**, que presentó una caída del 79% en ventas.
  
![Captura de pantalla_5-4-2026_221727_](https://github.com/user-attachments/assets/a9768b47-5d5e-4c6f-8951-4a3ab17a1974)

### 3. Insights Clave
* **Efecto "Audio":** Detecté que el bajo desempeño en regiones se debe a una dependencia de productos de baja rotación.
* **Potencial Capital:** Santiago mostró un crecimiento del +180%, validando la estrategia de expansión en la RM.
